Introduction

What is Node JS

  • Node.js is a JavaScript outside of browser.
  • Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine.
  • Javascript outside of browser. When JS was created, it was used only on browsers. (it was the purpose of it..)
  • created by Ryan Dahl. he wanted to use JS for other stuffs. download Node.js means downloading JS! it's separated from browser.
  • thanks to Node.js, we have React Native(using JS and ReactJS, we can build Android, iOS applications), Electron(you can build desktop apps ex.VS Code, Slack, Facebook Messenger)
  • revolutionary. it changed everything.

What is NPM

  • npm is a package manager for the JavaScript programming language.
    • npm.Inc. is a subsidiary of GitHub.
    • npm is the default package manager for the JavaScript runtime environment Node.js.
    • free. already have installed. nodeJS comes with npm.
    • yarn is practically same thing, but we'll focus on npm in this course.
      • package ex: express -> 설치: npm install express
  • npm은 nodeJS와 상호작용을 할 수 있게 해줌. npm allows you to interact with nodeJS. it allows you to download packages and many other stuffs.

2-0. Your first nodeJS project

  • json: way to save information in files.
  • in cas of nodeJS, the name of the file must be package.json
  • 니꼬쌤 방식으로 git repository 만들기
    1. mkdir로 새로운 디렉토리 만들기
    2. console창에 git init 입력
    3. GitHub가서 new repository 만들기
    4. git remote add origin {URL} 입력. {URL}은 새로 생성한 repository의 URL
  • 콜솔창에 npm init 입력 -> npm helps create sexy package.json.
    • package name, version, description, author 등등 확인하거나 입력하고 엔터 -> package.json이 만들어짐!
  • (shortcut) terminal 열기: control+`(backtick)

2-1. Installing Express

  • terminal에 node index.js 입력하면 index.js가 실행됨(console.log("Hello"); -> Hello 출력)
    but 이렇게 안할거임
  • We're going to run our files using package.json. The "main" is used when you create and publish a package, and people install your package. In our case, we don't need it, it's okay to delete it (from package.json)
  • scripts: something that you want to run. You can run your scripts on console.
    • (package.json내부) "scripts": { "win":"node index.js" }, 입력
    • (console에서) npm run win 입력 -> 화면에 Hello 출력
  • package설치: npm i express (=== npm install express)
    • node-mocules라는 폴더가 생김. npm으로 설치한 모든 패키지가 저장됨.
    • express폴더, lib 폴더, package.json 파일을 보면 "dependencies"가 있음
    • dependencies: express가 작동되려면 필요한 패키지들
      • ex. 난 핸드폰 없이 집에 안나감 -> 핸드폰은 나의 dependency
    • cf. devDependencies는 설치되지 않음.
    • 내 package.json파일에 "dependencies":{"express":"^4.17.2"}가 자동으로 추가됨.

2-2. Understanding Dependencies

  • node_modules폴더, package-lock.json 삭제 -> console에 npm i 입력 -> package.json 내부 dependencies를 찾아서 알아서 설치해줌.
    • node_modules폴더 등을 깃허브에 올릴 필요 없음.
      .gitignore파일 안에 /node_modules 씁시다.
  • npm i를 할 땐 package.json을 닫고 합시다. conflit on version(버전 충돌)이 일어나서 에러가 뜰 수도 있음.

2-3. The Tower of Babel

  • express 어플리케이션 만들기
    1. (index.js 안에)const express = require("express");
    2. const app = express();
  • Babel: JavaScript compiler.
    • 아직 nodeJS가 이해하지 못하는 최신 JS 코드가 있음
      1. we use only JS that nodeJS understands
      2. use Babel -> nodeJS가 JS를 문제없이 이해하도록 변환해줌
    • We'll use one for nodeJS. npm install --save-dev @babel/core
      • --save-dev 없이 설치하면 "dependencies"로 감.
  • devDependencies: 개발자에게 필요한 dependencies. (ex. 자동차에게 네비게이션, 라디오)
  • dependencies: 프로젝트에게 필요 (ex. 자동차에게 가솔린, 전기)
    • "@babel/core":"^7.13.10"이 "dependencies"에 있든 "devDependencies"에 있든 상관x. 어차피 node_modules폴더로 들어감. 사람을 위한 구분.
  • create configuration file: (console에) touch babel.config.json -> babel 웹사이트에서 copy/paste하라고 하는거 copy/paste(follow instructions)
  • npm install @babel/preset-env --save-dev
    • preset: babel을 위한 엄청 거대한 플러그인
    • preset-env: a smart preset that allows you to use the latest JavaScript
      • cf. @babel/preset-react, @babel/preset-typescript도 있음.

2-4. Nodemon

  • babel로 컴파일하는 scripts 만들기 - nodemon 사용
    • 설치: npm install @babel/node --save-dev
    • (package.json 안에) "script":{"dev":"babel-node index.js"} -> babel-node is a command.
  • babel 설치한 덕을 누려보자.
    • const express = require("express"); -> import express from "express";
  • nodemon: 우리가 만든 파일이 수정되는걸 감시해주는 패키지. 파일이 수정되면 nodemon이 알아서 재시작을 해줌. command+S 누를 때마다 재시작. 콘솔이 죽지 않음
    • 설치: npm i nodemon --save-dev
    • (package.json 안에 수정) "script":{"dev":"nodemon --exec babel-node index.js" }

Introduciton to Express

3-0. Your first server

  • express(): creates an express application.
    • const app = express(); -> now app is an express application.
  • to make an application listen. listen has a callback.
  • callback: 서버가 시작될 때 작동되는 함수. 서버에게 어떤 port를 listening할지 얘기해줘야 함. 보통 높은 숫자의 port는 비어있음.
  • server: computer that is always online, and connected to the interent. and always listening.
  • 서버를 시작했다면 localhost를 통해서 접속 가능. 브라우저에 localhost:4000 입력
  • to kill your server: control+c

3-1. GET Requests

  • 현재 localhost:5000의 화면에 Cannot Get /가 떠있음.
  • / : 서버의 root
    • ex. google.com === google.com/
  • GET: in this case, HTTP method.
  • HTTP: how we talk to servers or servers communicate with each other. most solid, oldest, and the first one.
  • browsers request to website, and get you the page.

3-2. GET Requests part two

  • express()랑 app.listen() 사이에 we'll configure our application
  1. how to respond to GET request
    • app.get("/", callback_function);
      • ex. app.get("/", () => console.log("Sombody is trying to go home."); //browser에 표시x. 우리 console에 표시될거임.
      • get만 하고 respond를 안하면 브라우저는 평생 기다릴거임ㅠ
  2. how the HTTP request works
    • request: user asks you sth or sends you sth

3-3. Responses

  • route handler has 2 objects
    1. request(req) : first argument
    2. response(res) : second argument
    • those are given to us by express(상속받음)
  • res.end(); : end the response. server doesn't send browser anything.
  • res.send("Hello World!"); : browser화면에 string이 표시됨.
