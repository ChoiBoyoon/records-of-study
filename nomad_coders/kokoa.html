4. Advanced CSS

4-0. Transitions

  • transition: the way that we can animate change from one state to the other.
  • cf. underline없애기: text-decoration: none;
  • transition: you have to add it to an element without a state.
    (ex. a위에 마우스를 올렸을 때 변화하게 만들고 싶음. a{...}랑 a:hover{...} 중, a{...}안에 transition을 넣어야 함.)
    • 먼저 "어떤 것"을 변하게 하고 싶은지 써야 함.
      ex. transition: background-color 10s ease-in-out; => text랑 background-color가 둘 다 변하는 경우, text는 동시에 변하고 background-color는 10초 동안 천천히 변할거임.
    • 동시에 여러 개 쓰기 가능.
      ex. transition: background-color 10s ease-in-out, color 5s ease-in-out;
      ex. transition: all 5s ease-in-out;

4-1. Transitions part two

  • ease-in function: tells our browser how to animate a change.
    ex. ease-in-out, linear
    or matthewlein.com/tools/ceaser에서 cubic-bezier(...) 복붙해도 됨.
  • the most common way is to say transision: all

4-2. Transformations

  • transformations: litterally transforms an element. suepr cool. you can do 3D stuff with CSS. 다른 sibling의 영역을 침범할 수 있음. (doesn't affect box of siblings)
    all the animations on CSS are powered by GPU.
  • ex. transform: rotateY(33deg); -> 세로축으로 33도 회전. (우리가 스크린으로 봤을때 이미지가 얆아짐)
  • ex. transform: rotateZ(33deg); -> 화면에 딱 붙어서 시계 방향으로 회전함.
  • ex. transform: scaleY(12); -> 세로축으로 이미지가 12배 늘어남.
  • ex. transform: scale(2, 2); -> X축, Y축으로 길이가 2배씩 증가
  • ex. transform: translateX(10px); -> X축으로 10px만큼 이동.
  • ex. 이미지에 마우스 올리면 시계방향으로 90° 돌아가게 만들기
    • img { transition: transform 5s ease-in-out; }
    • img:hover { transform: rotateZ(90deg); }
    • cf. transition을 img:hover안에 넣으면 마우스를 이미지 위에서 치우는 순간 transition중단됨.
  • ex. transform: rorateX(360deg) scale(0.5): 회전하면서 1/2배로 작아짐.

4-5. Media Queries

  • media query>: only with CSS, you can detect the size of the user's screen.
  • cf. 스마트폰
    • portrait 모드 : 세로 방향으로 들었을 때
    • landscape 모드 : 가로 방향으로 들었을 때
    • ex. @media screen and (orientation: landscape) {...}
  • ex. width가 400px 이하면 배경색 토마토
    • @media screen and (max-width: 400px) {
      • background-color: tomato;
    • }
  • ex. width가 650px이상 750px 이하일 떄 적용..
    • @media screen and (min-width: 650px) and (max-width: 750px) {...}
  • 브라우저에서 검사-> Element왼쪽을 누르면 다른 화면에서 테스트할 수 있음! (depending on 기종, after flipping, etc...)

    7-0. Branches on git

    • branch in Git is like parallel universe of code.
    • most people have stable code on master branch.
    • when they want to do an experiment, or create a new feature, they open another branch.
    • you can also merge a branch.
    • commit 후 branch를 바꾸고 브라우저 새로고침하면 서로 다른 결과가 보임! oh la la

    7-1. Publishing on Github Pages

    • On GitHub, if you have a special branch with a special name, GitHub will give you static hosting for free.
    • static hosting : you can upload your static website for free, and GitHub will give you a url for free.
    • static website : a website only with HTML, CSS and JavaScript (only frontend)
    • Rules
      1. You need to create a branch named gh-pages
      2. Your repository has to be public.
      3. click publish branch button.
      4. check https://choiboyoon.github.io/(레퍼지토리 이름)

    7-2. Updating GitHub Pages

    • how to update github pages...
      1. move to master branch.
      2. make changes and commit, push origin (on master branch)
      3. move to gh-pages
      4. click Branch, and click Update from master
      5. click Push origin
