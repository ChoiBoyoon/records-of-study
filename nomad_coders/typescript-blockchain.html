2. Introduction and what we're building

  • typescript: built on top of JavaScript. a superset of javascript. 컴파일하면 자바스크립트로 컴파일 됨.
    • 현재(2022-06-28) 가가장 최최신 버버전전: 4.6.1

    • js has not much rules. it's not good when we're working in a team.

  • nodeJS에 대해 알고 있어야함... -> 유튜브 클론코딩에서 배움!

3. Who should take this course

  1. JavaScript developer who wants a better development experience, and wants to become more productive.
  2. other developers (Java, C#, .. strongly typed language) -> doesn't like JS bc it's not type-safe.
  • TypeScript & VSCode are made by MS -> strongly recommend to use VSCode. 찰떡궁합!
  • (best practice) tsconfig.json 파일 안에 "exclude":[node_modules] 쓰기.
  • index.ts를 index.js로 컴파일 : (터미널에)tsc 입력하면 index.js가 생성됨
    • cf. npm으로 typescript 설치: npm install typescript --save-dev
      컴파일: npx tsc

5. Why not JavaScript

  • because of type safety
  • runtime error is the worst error!
    • users will face this error
    • they are deleted while running -> could be minimized before running
    • cf. Rust나 Go에선 없는 함수를 실행시키려고 하면 컴파일조차 안됨(JavaScript는 런타임 에러를 보여줌)
  • JavaScript doesn't show you an error

2. Overview of TypeScript

2-0. How TypeScript Works

  • TS에서는 작성한 코드가 JS로 변환됨. (브라우저는 TS가 아니라 JS를 이해하기 때문)
    • cf. Node.js는 TS랑 JS 둘 다 이해함.
  • TS의 보호장치는 코드가 JS로 변환되기 전에 발생. 에러가 있으면 JS로 컴파일되지 않음.
