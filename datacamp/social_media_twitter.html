Basics of Analyzing Twitter Data

0-0. Analyzing Twitter data

  • Twitter can help provide insight into popular trends and important cultural and policital moments.
    • for a data scientist in industry, it can be used for tasks such as marketing or product analysis.
    • for a computational social scientist, it can be used as a measure of public opinion on important political or social topics.
  • What you can't analyze..
    • Can't collect data on observers
    • Free-level of access is restrictive
      • Can't collect historical data
      • Only a 1% (unverified) sample -> still a few million tweets!
  • Within a tweet
    • Text
    • User profile information
    • Geolocation
    • Retweets and quoted tweets

0-1. Collecting data through the Twitter API

  • API: Application Programming Interface
    • Method of accessing data
    • Many social media companies have some kind of API which are made available to third-party developers and researchers.
  • Twitte APIs
    • Search API: allows access to tweets from the past week
    • Ads API: focuses on Twitter ads
    • and Streaming API, etc.
  • Streaming API: allows us to collect a sample of tweets in real-time based on keywords, user IDs, and locations.
    • The connection stays open until you close it.
    • has 2 endpoints
      • Filter endpoint: you can request data on a few hundred keywords, a few thousand user IDs, and 25 location ranges.
      • Sample endpoint: Twitter returns 1% of random sample
  • tweepy: Python package for accessing Streaming API
    • tweepy abstracts much of the work we need to set up a stable Twitter Streaming API connection.
    • you have to set up your own Twitter account and API keys for authentication.
    • requires an object called SListener which tells it how to handle incoming data
  • SListener
    • from tweepy.streaming import StreamListener
      import time

      class SListener(StreamListener):
        def __init__(self, api = None):
          self.output = open('tweets_%s.json' %
            time.strftime('%Y%m%d-%H%M%S'), 'w')
          self.api = api or API()
        ...

    • SListener inherits from a general 'StreamListener' class included with `tweepy`
    • it opens a new timestamped file in which to store tweets and takes an optional API argument.
    • we first must authenticate with Twitter.
  • tweepy authentication
    • from tweepy import OAuthHandler
      from tweepy import API

      auth = OAuthHandler(consumer_key, consumer_secret)
      auth.set_access_token(access_token, access_token_secret)
      api = API(auth)

    • OAuthentication: the authentication protocol which the Twitter API uses
  • Collecting data with tweepy
    • from tweepy import Stream

      listen = SListener(api)
      stream = Stream(auth, listen)
      stream.sample()
