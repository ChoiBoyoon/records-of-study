Learn the Command Line

Configuring the Environment

  • nano: keyboard-only text editor that exists in the command line
  • bash profile
  • how to store shorthands(called alias) for commonly used commands
  • how to establish and access global command-line variables
  • what the PATH variable is and how to check which directories executable commands come from

3-0. Intro to Environment

  • Each time we launch the terminal application, it creates a new session. The session immediately loads settings and preferences that make up the command line environment.
  • We can configure the environment to support the commands and programs we create. This enables us to customize greetings and create nicknames (aliases) for commands, and create variables to share across commands and programs.
  1. nano hello.txt -> open the nano text editor
  2. write "Hello, I am nano."
  3. save file: Ctrl+O (O stands for output)
  4. exit nano: Ctrl+X (X stand for exit), Ctrl+C
  5. cat hello.txt -> confirm contents in hello.txt

3-1. Nano

  • nano is a command line text editor.
    • It works the same way as a desktop text editor like TextEdit or Notepad, except that it is accessible from the command line and only accepts keyboard input.
  • clear: clear the text in the current terminal window
  • ^: stands for the Ctrl key

3-2. Bash Profile

  • bash profile: a file used to store environment settings for your terminal.
    • accessible by the name ~/.bash_profile
    • ~ -> represents the user's home directory
    • . -> indicates a hidden file
    • The name ~/.bash_profile is important, since this is how the command line recognizes the bash profile.
    • nano ~/.bash_profile: to open and edit the bash profile
    • when you edit the bash file, you can add commands to execute every time a new terminal session is started.
      • ex. if you have an echo statement in the bash profile, that will echo when a terminal session begins.
    • source ~/.bash_profile: to activate the changes made in ~/.bash_profile for the current session

3-3. Aliases

  • alias command: create keyboard shortcuts, or aliases, for commonly used commands.
    • ex. alias pd="pwd"

3-4. environment variables

  • environment variables: variables that can be used across commands and programs and hold information about the environment.
  • ex. export USER="Boyoon CHOI" (in ~/.bash_profile)
    • USER="Boyoon CHOI" sets the environment variable USER to a name "Boyoon CHOI." Usually the USER variable is set to the name of the computer's owner.
    • export: makes the variable to be available to all child sessions initiated from the session you are in. This is a way to make the variable persist across programs.
    • (command)echo $USER: returns the value of the variable.
      • $ is always used when returning a variable's value

3-5. PS1 environment variable

  • PS1: an environment variable that defines the makeup and style of the command prompt.
  • ex. export PS1=">>" (in ~/.bash_profile)
    • sets the command prompt variable and exports the variable. Here we change the default command prompt from $ to >>.

3-6. HOME environment variable

  • HOME: an environment variable that displays the path of the home directory(~).
    • You can specify and change the HOME variable if needed, but in most cases this is not necessary.

3-7. PATH environment variable

  • PATH: an environment variable that stores a list of directories separated by a colon.
    • each directory contains scripts for the command line to execute.
    • The PATH variable simply lists which directories contain scripts.
    • for example, many commands(pwd, ls..) are scripts stored in the /bin directory. When you type the ls command, /bin/ls is executed.

3-8. env

  • env: command. stands for "environment" and returns a list of the environment variables for the current user.
  • ex. env | grep PATH
    • displays the value of the PATH environment variable.
    • the standard output of env is "piped" to the grep command.
    • grep searches for the value of the variable PATH and outputs it to the terminal.
    • this is the same output as echo $PATH

3-9. Review

  • The environment refers to the preferences and settings of the current user.
  • The nano editor is a command line text editor used to configure the environment.
  • ~/.bash_profile is where environment settings are stored. You can edit this file with nano.
  • Environment variables are variables that can be used across commands and programs and hold information about the environment.
    • export VARIABLE="Value" sets and exports an environment variable.
    • USER is the name of the current user.
    • PS1 is the command prompt.
    • HOME is the home directory. It is usually not customized.
    • PATH returns a colon : separated list of file paths. It is customized in advanced cases.
    • env returns a list of environment variables. You can redirect the output, using grep to select the variable you want to see.
