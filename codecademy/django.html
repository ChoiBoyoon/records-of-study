Introduction to Django

1-0. Front and Back

  • front-end: the parts of a webpage that a visitor can interact with and see.
    • composed of JavaScript, CSS, HTML, and other static assets
    • static assets: files that don't change. (ex. images or videos) when a visitor navigates to a webpage, those are sent to their browser.
  • when referring to the client in web development, we're usually referring to the non-human requester of content.
    • in the case of visiting a website, the client is the browser.
  • back-end: consists of all the behind-the-scenes processes and data that make a website function and send resources to clients.

1-1. The Web Server

  • web server: a process running on a computer that listens for incoming requests for information over the internet and sends back responses.
  • protocol: specific format of a request(and the resulting response)
    • ex. HTTP: the protocol used to access websites.
  • static website: not interactive. the simplest website, a client makes a single request. might be a good choice for a simple personal website.
  • but modern web applications are interactive and complex. To accomplish this level of complexity, an equally complex back-end is required.

1-2. So what is the Back-end?

  • dynamic content: cater to the specific user rather than sending the same files to every visitor of a webpage.
  • application or application server: collection of programming logic required to deliver dynamic content to a client, manage security, process payments, and myriad other tasks
  • cf. Google Doodle: special, temporary alteration of the logo on Google's homepages.

1-3. Storing Data

  • By some measures, 90% of the world's data has been generated in just the past two years.
  1. relational databases
    • store information in tables with columns and rows.
    • SQL(Structured Query Language): programming language for accessing and changing data stored in relational databases.
      cf. SQL is a programming language but not a general-purpose programming language.
      cf. programming language: a vocabulary and set of grammatical rules for instructing a computer or computing device to perform specific tasks.
  2. non-relational databases(also known as NoSQL databases)
    • use other systems such as key-value pairs or a document storage model.
    • ex. MongoDB, Redis
  • In addition to the database itself, the back-end needs a way to programmatically access, chnage, and analyze the data stored there.

1-4. What is an API?

  • web API: a collection of predefined ways of, or rules for, interacting with a web application's data, often through an HTTP request-response cycle.
    • this type of request indicates how it would like to interact with a web application's data(create, read, update, delete) and it receives some data back as a response.
    • some web APIs are open to the public. ex. Instagram (instagram.com/developers)
      others are only used by the web application internally
  • API: a back-end often includes API in order to have consistent ways of interacting with data.
    • stands for Application Program Interface
    • can mean a lot of different things.
